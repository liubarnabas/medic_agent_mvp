# 项目概述

**项目名称：** 临床辅助诊断智能体（Clinical Diagnostic AI Agent）
**MVP 范围：** 缺铁性贫血（IDA）诊断通路
**生成日期：** 2026-03-16

---

## 产品定位

面向临床医生的**辅助诊断工具**，基于国家标准（WS/T 405）和临床诊疗指南，对检验数据进行结构化推理，输出具备诊断级别精度的辅助结论。

**核心设计原则：可解释、可审计、可复制。**

---

## 技术栈总览

| 层 | 技术 | 职责 |
|----|------|------|
| 前端 | React 18 + Vite 5 | 数据输入表单 + 诊断结果展示 |
| 后端 API | Python FastAPI + Uvicorn | HTTP 接口，协调计算/检索/融合三链路 |
| 规则引擎 | 纯 Python（Pydantic） | IDA 条款 0~9 判断，无 LLM，可解释 |
| LLM 层 | Anthropic SDK | 检索链路 + 自然语言诊断描述生成 |
| 测试 | pytest | 规则引擎单元测试 |

---

## 系统架构类型

**多部分架构（Multi-Part）**：
- `frontend`：React SPA，端口 5173（开发）
- `backend`：FastAPI，端口 8000

两部分通过 REST API（`/api/diagnose`）通信。

---

## 核心业务逻辑

系统采用"计算链路 + 检索链路 + 融合决策"三层架构：

| 链路 | 实现 | MVP 状态 |
|------|------|----------|
| 计算链路 | `src/ida/engine.py`（纯规则） | ✅ M1 核心 |
| 检索链路 | Anthropic SDK（LLM 直接调用） | M4 |
| 融合决策 | `server.py`（一致性比对） | M5 |

---

## 诊断产物

MVP 对 IDA 输出以下结构化结论：

- **确诊**：小细胞低色素 + 条款 2~9 中 ≥ 2 条阳性
- **高度提示**：SF < 14 μg/L（未达确诊标准）
- **阴性**：证据不足，不能诊断
- **不适用**：存在前置条件（妊娠、肝硬化等血容量变化情况）

---

## MVP 里程碑

| 里程碑 | 目标 | 优先级 |
|--------|------|--------|
| **M1** | 编码实现 IDA 计算链路（纯规则引擎） | ⭐ 高 |
| **M2** | 用 10~20 个真实病例验证规则覆盖度 | ⭐ 高 |
| **M3** | 拿结果找检验科医生确认 | ⭐ 高 |
| M4 | 检索链路用 LLM 直接跑 | 中 |
| M5 | 融合决策联调与报告输出 | 中 |
| M6 | 第二个病种模板填充与验证 | 中 |
