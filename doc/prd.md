## 1. 产品概述

### 1.1 产品定位

本系统是一款面向临床医生的**辅助诊断工具**，而非体检报告解读工具。系统基于国家标准、临床诊疗指南和循证医学证据，对检验数据进行结构化推理，输出具备诊断级别精度的辅助结论。

核心设计原则：**可解释、可审计、可复制**。

### 1.2 核心架构思路

系统采用 **"计算链路 + 检索链路 + 融合决策"** 三层架构：

- **计算链路**：基于指南的确定性规则推理引擎，输入检验 JSON，输出结构化诊断结论。
- **检索链路**：基于大语言模型（LLM）的相似病例检索与 Top-N 诊断概率排序。
- **融合决策**：对两条链路的结论进行一致性比对，一致则高置信输出，不一致则降级并提请人工复核。

### 1.3 最小可行产品（MVP）范围

MVP 以**缺铁性贫血（IDA）** 作为首个诊断通路。选择理由：

- 标记物少、指南明确、争议小
- 属于教科书级确定性推理
- 可验证整个架构模式的可行性

## 2. 背景与目标

### 2.1 业务背景

检验医学报告的解读高度依赖医生个人经验。在基层医疗机构，由于专科医生不足，检验报告的诊断潜力未被充分利用。同时，成熟的临床诊疗指南为自动化推理提供了坚实的知识基础。

### 2.2 产品目标

| 目标层级 | 描述 |
|----------|------|
| 短期目标 | 以 IDA 为案例跑通"计算+检索+融合"全链路，验证模式可行性 |
| 中期目标 | 扩展至检验医学更多病种（巨幼细胞贫血、再生障碍性贫血、慢性病贫血等） |
| 长期目标 | 横向扩展至流式细胞术、血液科等其他专业，实现多专科覆盖 |
| 远期愿景 | 积累结构化诊断数据，支持从"辅助诊断"向"疾病风险预测"演进 |

### 2.3 成功标准

- IDA 诊断通路对 20 例真实病例的覆盖率 ≥ 95%
- 计算链路输出结论与临床医生金标准判断的一致率 ≥ 90%
- 融合决策中"高置信"占比 ≥ 70%
- 所有医生对输出报告的可接受度评分 ≥ 4/5

## 3. 用户画像与使用场景

### 3.1 目标用户

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| 检验科医生 | 出具检验报告的一线医生 | 对异常指标组合快速给出诊断方向提示 |
| 临床主治医生 | 收到检验报告后做诊疗决策的医生 | 快速确认/排除诊断，获取鉴别诊断建议和补充检查建议 |
| 基层全科医生 | 专科知识有限的基层医生 | 获得专科级别的诊断辅助，减少误诊漏诊 |

### 3.2 临床场景分层

系统识别三大使用场景，MVP 优先覆盖场景 B：

| 场景 | 类型 | 描述 | 系统任务 | 优先级 |
|------|------|------|----------|--------|
| 场景 A | 体检/筛查型 | 患者无症状，拿到检验结果想知道有没有问题 | 异常检出 → 可能诊断推理 | 后续版本 |
| **场景 B** | **症状导向型** | **医生已有方向，需要检验证据确认** | **证据匹配 + 确认/排除** | **⭐ MVP 优先** |
| 场景 C | 复杂/排除型 | 症状无特异性，需广撒网排除严重疾病 | 逐一排除 + 风险标记 | 后续版本 |

缺铁性贫血正好对应场景 B——医生已怀疑贫血，用检验结果确认类型和分期。
场景A也适用，因为大多数IDA患者症状不明显，可能是从注意力不集中或者精神不好、乏力等来体检发现
场景C也部分适用，因为IDA可以和其他疾病混杂共存，比如IDA伴随MA可能MCV并不会低

### 3.3 典型用户故事

**US-01：** 作为一名临床医生，我已怀疑患者为缺铁性贫血，我输入患者的血常规和铁代谢指标，希望系统帮我确认诊断、给出分期和严重程度，并提示需要做的鉴别诊断和补充检查。

**US-02：** 作为一名基层全科医生，我看到一个患者 Hb 偏低但 SF 正常，我不确定是否为缺铁性贫血，希望系统给出分析和下一步建议。

**US-03：** 作为一名检验科医生，我想批量复核今天出具的贫血相关报告，希望系统对每份报告给出结构化的辅助诊断结论。

## 4. 系统架构设计

### 4.1 总体架构

```
┌─────────────────────────────────────────────────────────┐
│                      输入层                              │
│  患者基本信息 + 检验指标 JSON + 临床信息（可选）           │
└──────────────────────┬──────────────────────────────────┘
                       │
          ┌────────────┴────────────┐
          ▼                         ▼
┌──────────────────┐     ┌──────────────────┐
│   计算链路        │     │   检索链路        │
│ （确定性规则引擎） │     │ （LLM + 病例库）  │
│                  │     │                  │
│ 指南规则编码      │     │ 相似病例匹配      │
│ 阈值判定         │     │ Top-N 诊断排序    │
│ 分期/分级        │     │ 指南条目检索      │
│ 鉴别诊断触发     │     │                  │
└────────┬─────────┘     └────────┬─────────┘
         │                        │
         └───────────┬────────────┘
                     ▼
          ┌──────────────────┐
          │   融合决策层      │
          │                  │
          │ 一致性比对        │
          │ 置信度评定        │
          │ 冲突标记          │
          └────────┬─────────┘
                   ▼
          ┌──────────────────┐
          │   输出层          │
          │                  │
          │ 结构化诊断结论    │
          │ LLM 自然语言描述  │
          │ 辅助诊断报告      │
          └──────────────────┘
```

### 4.2 计算链路设计要点

- **纯规则引擎**：输入 JSON，输出 JSON，不依赖 LLM，确保可解释性和确定性。
- **参考区间动态更新**：依据 WS/T 405 最新版本，系统每年自动更新参考下限。
- **证据可追溯**：每条判定结果均附带检验值、参考标准及标准年份。
- **缺失值容错**：未检查/未开展的项目不计分、不扣分，而非报错。

### 4.3 检索链路设计要点

- 输入同一份标准化报告，LLM 检索相似病例（铁代谢指标模式匹配）和相关指南条目。
- 输出 Top-2/3 可能诊断及证据。
- MVP 阶段先用 LLM 直接跑，不需先建病例库。

### 4.4 融合决策逻辑

| 条件 | 结果 | 输出行为 |
|------|------|----------|
| 计算链路 = IDA **且** 检索 Top-1 = IDA | ✅ 高置信度 | 输出 IDA 诊断，附分期和程度 |
| 计算链路有冲突标记 **或** 检索 Top-1 不一致 | ⚠️ 降级 | 需人工复核，列出分歧点 |
| 关键指标缺失 | ❗ 提示 | 建议补充指定检查项目 |

### 4.5 多诊断并行处理策略

"一个病人可能同时有几十种病"——系统最终必须支持多诊断并行推理。但在 MVP 中：

- **主干**只走单一诊断通路（如单纯缺铁性贫血的诊断，不包括非单纯性的缺铁性贫血如IDA伴随MA或者慢性病贫血或者地贫等复杂情况）。
- **旁路提示**：发现与当前诊断通路无关但异常的指标时，只做标记，不做诊断，不做推理，可做根据AI的临床意义提示，但是要标注清楚不作为诊疗意见。
- 示例输出：*"肝功能 ALT 轻度升高，非本通路，建议另行评估"*

### 5.2 F-01：患者信息与检验数据输入

**功能描述：** 系统接收患者基本信息和检验指标数据作为诊断输入。

**输入格式：** JSON

**必需字段：**

| 类别 | 指标 | 字段名 | 单位 | 必要性 |
|------|------|--------|------|--------|
| 基本信息 | 性别 | `sex` | "male" / "female" | 必须 |
| 基本信息 | 年龄 | `age` | 岁 | 必须 |
| 基本信息 | 是否妊娠 | `pregnant` | boolean | 女性必须 |
| 基本信息 | 前置条件标志 | `precondition` | boolean | 必须 |
| 血常规 | 血红蛋白 | `Hb` | g/L | 必须 |
| 血常规 | 平均红细胞体积 | `MCV` | fL | 必须 |
| 血常规 | 平均红细胞血红蛋白含量 | `MCH` | pg | 必须 |
| 血常规 | 平均红细胞血红蛋白浓度 | `MCHC` | 0~1 或 g/L | 必须 |
| 血常规 | 红细胞分布宽度 | `RDW` | % | 建议 |
| 铁代谢 | 血清铁蛋白 | `SF` | μg/L | 必须 |
| 铁代谢 | 血清铁 | `serum_iron` | μmol/L | 建议 |
| 铁代谢 | 总铁结合力 | `TIBC` | μmol/L | 建议 |
| 铁代谢 | 转铁蛋白饱和度 | `TS` | 0~1 | 必须 |
| 炎症指标 | CRP | `CRP` | mg/L | 建议 |
| 炎症指标 | ESR | `ESR` | mm/h | 可选 |
| 临床信息 | 病因列表 | `etiology` | dict | 可选 |
| 临床信息 | 症状/体征列表 | `symptoms` | dict | 可选 |

**可选扩展字段：**

| 指标 | 字段名 | 单位 | 说明 |
|------|--------|------|------|
| 红细胞游离原卟啉 | `FEP` | μmol/L | 部分医院开展 |
| 血液锌原卟啉 | `ZPP` | μmol/L 或 μg/g Hb | 部分医院开展 |
| 可溶性转铁蛋白受体 | `sTfR` | nmol/L | 部分医院开展，界限值与设备有关 |
| 骨髓铁染色结果 | `bone_marrow_iron_stain` | 枚举 | 部分医院开展 |
| 铁剂治疗后 Hb | `Hb_after_iron_therapy` | g/L | 治疗 2~4 周后复查值 |

**单位换算规则（系统内置）：**

- 血清铁：50 μg/dL ≈ 8.95 μmol/L
- 总铁结合力：360 μg/dL ≈ 64.44 μmol/L
- FEP：50 μg/dL ≈ 0.9 μmol/L
- ZPP：60 μg/dL ≈ 0.96 μmol/L
- sTfR：2.25 mg/L ≈ 26.5 nmol/L（R&D systems）

### 5.3 F-02：前置条件排除判定

**功能描述：** 判断患者是否存在血容量变化等前置条件，若存在则不适用自动诊断流程。

**排除条件（任一满足即排除）：**

- 妊娠
- 肝硬化
- 低蛋白血症
- 充血性心力衰竭
- 脱水
- 大剂量利尿剂
- 急性大出血（外伤、消化道大出血等）

**输出：** `"不适用后续流程，建议去医院找专业医生判断"`

### 5.4 F-03：贫血判定

**功能描述：** 根据 Hb 值与国家标准参考下限比较，判定是否存在贫血。

**参考标准：** WS/T 405 最新版本（系统每年自动更新）

**判定规则：**

| 人群 | Hb 参考下限 | 判定 |
|------|-------------|------|
| 成年男性 | 120 g/L | Hb < 120 → 贫血 |
| 成年女性（非孕） | 115 g/L | Hb < 115 → 贫血 |
| 孕妇 | 110 g/L | Hb < 110 → 贫血 |

> **注：** WS/T 405-2012 标准值为男性 120 g/L、女性 115 g/L。WHO 标准（男 130、女 120、孕妇 110）作为备用参考，系统可配置。

**干扰因素提示（Hb ≥ 参考下限时输出）：**

当前不满足贫血诊断，但提示医生排除以下干扰因素后重新人工评估：

- 高海拔地区长期居住（≥ 6~12 个月）
- 长期慢性呼吸道疾病（慢阻肺、长期吸烟、呼吸睡眠暂停等）
- 影响 Hb 检测结果的其他因素

### 5.5 F-04：贫血形态学分类

**功能描述：** 根据 MCV、MCH、MCHC 判定贫血形态学类型。

| 分类 | 条件 | 临床提示 |
|------|------|----------|
| 小细胞低色素性贫血 | MCV < 80 fL **且** MCH < 27 pg **且** MCHC < 0.32 | 提示缺铁 / 地贫 / 慢病贫血 |
| 正细胞正色素性贫血 | MCV 80~100 fL | 提示急性失血 / 慢病 / 混合 |
| 大细胞性贫血 | MCV > 100 fL | 提示 B12/叶酸缺乏 / MDS |

IDA 确诊要求必须满足小细胞低色素性贫血条件（或在特别提示路径中单独由 SF 触发）。

### 5.6 F-05：贫血程度分级

| 程度 | Hb 范围 |
|------|---------|
| 轻度 | 90 g/L ~ 正常下限 |
| 中度 | 60 ~ 90 g/L |
| 重度 | 30 ~ 60 g/L |
| 极重度 | < 30 g/L |

### 5.7 F-06：铁缺乏支持证据累计与确诊判定

**功能描述：** 对条款 2~9 逐一评估，每满足一条记 1 分，满足 ≥ 2 条即确诊 IDA。

**条款定义：**

| 条款 | 判断内容 | 阳性条件 |
|------|----------|----------|
| 条款 2 | 病因 + 体征 | 至少 1 项明确缺铁病因 **且** 至少 1 项临床体征 |
| 条款 3 | 血清铁 + TIBC | 血清铁 < 8.95 μmol/L **且** TIBC > 64.44 μmol/L （未检查则跳过）|
| 条款 4 | 转铁蛋白饱和度 | TS < 0.15（即 < 15%）（未检查则跳过） |
| 条款 5 | 骨髓铁染色 | 外铁阴性~弱阳 **且** 铁粒幼红细胞 < 15%（未检查则跳过） |
| 条款 6 | FEP / ZPP | FEP > 0.9 μmol/L **或** ZPP > 0.96 μmol/L **或** ZPP > 3.0 μg/g Hb |
| 条款 7 | 血清铁蛋白 | SF < 14 μg/L（⚠️ 同时用于特别提示路径单独判定） |
| 条款 8 | sTfR | sTfR > 26.5 nmol/L（R&D systems；其他设备界限值不同，未开展则跳过） |
| 条款 9 | 铁剂治疗反应 | 治疗 2~4 周后 Hb 升高 > 15 g/L（未实施则跳过） |

**缺铁病因枚举（条款 2）：**

- `absorption_disorder` — 吸收不良（胃/十二指肠/空肠近端手术或疾病）
- `vegetarian_picky_eating` — 素食/偏食
- `adolescent_growth` — 生长发育期（青春期）
- `pregnancy` — 妊娠期
- `multiparity` — 多次妊娠（≥ 3 胎）
- `chronic_bleeding` — 慢性出血（消化道出血、月经过多等）
- `other_evidenced` — 其他病因（系统初始化时由 Agent 检索循证证据后补全）

**临床体征枚举（条款 2）：**

- `melena` — 黑便
- `menorrhagia` — 经血过多
- `chronic_diarrhea_constipation` — 慢性腹泻或便秘
- `fatigue` — 疲劳、倦怠、乏力
- `pallor` — 皮肤、黏膜和甲床苍白
- `tachycardia_etc` — 心动过速、心悸、心绞痛、头晕目眩、注意力下降
- `dyspnea_on_exertion` — 呼吸加深（运动/情绪激动时更明显）
- `koilonychia` — 反甲/勺状指
- `dry_skin` — 皮肤干燥
- `angular_cheilitis` — 口角炎、舌炎
- `brittle_hair` — 头发干枯易断
- `pica` — 偏食、异食癖

**计分规则：**

- `None`（未检查/未开展）：不计入阳性数，也不计入总数
- `True`：阳性，计 1 分
- `False`：阴性，不计分

**判定结果：**

- 阳性数 ≥ 2 **且** 小细胞低色素 → **确诊缺铁性贫血（IDA）**
- 阳性数 < 2 → 进入特别提示路径
条款5和条款9也高度疑似缺铁性贫血

### 5.8 F-07：特别提示路径

**功能描述：** 当无法确诊 IDA 时（包括非小细胞低色素性贫血的情况），对 SF 进行单独判定。

**逻辑：**

- SF 有值 **且** SF < 14 μg/L → 输出：*"血清铁蛋白 < 14 μg/L，高度提示缺铁性贫血，请结合其他检查进一步确认。"*
- SF ≥ 14 μg/L **或** SF 为空值 → 输出：*"证据不足，不能诊断缺铁性贫血。建议补充检查 / 随访 / 试验性铁剂治疗后评估。"*

### 5.9 F-08：缺铁性贫血分期

| 分期 | 铁代谢特征 | Hb 状态 | MCV 状态 |
|------|------------|---------|----------|
| 铁减少期（ID） | SF↓ | 正常 | 正常 |
| 缺铁性红细胞生成期（IDE） | SF↓，TS%↓ | 正常或轻度↓ | 正常或轻度↓ |
| 缺铁性贫血期（IDA） | SF↓，TS%↓ | Hb↓ | MCV↓ |

### 5.10 F-09：鉴别诊断触发

| 触发条件 | 需排除疾病 | 系统建议 |
|----------|------------|----------|
| 小细胞低色素 + SF 正常/升高 | 地中海贫血 | 建议血红蛋白电泳 |
| 小细胞低色素 + CRP 升高 | 慢性病贫血 | 结合 CRP/ESR 综合判断 |
| 小细胞低色素 + SF 正常 + CRP 正常 | 铁粒幼细胞贫血 | 建议骨髓铁染色 |
| SF↓ + MCV 正常/升高 | 混合性贫血 | 同时缺铁 + B12/叶酸，建议补查 |

### 5.11 F-10：病因提示

| 人群特征 | 提示内容 |
|----------|----------|
| 育龄女性 | 月经过多？ |
| 老年男性 | 消化道失血？建议便潜血/胃肠镜 |
| 儿童 | 饮食/生长发育？ |

> 注：病因提示为非诊断性提示，仅供参考。

### 5.12 F-11：冲突检测与降级处理

| 编号 | 冲突条件 | 处理方式 |
|------|----------|----------|
| 冲突 1 | Hb 正常但 SF < 15 μg/L | 非贫血但铁缺乏，提示"铁减少期" |
| 冲突 2 | MCV↓ + SF 正常 + CRP 正常 | 不典型，建议补查 Hb 电泳 或地贫相关基因|
| 冲突 3 | 关键指标缺失 | 标记缺失项，降级置信度 |

## 7. 核心诊断逻辑——缺铁性贫血（IDA）

### 7.1 诊断路径总览

```
是否满足「前置条件」？（妊娠、肝硬化、低蛋白血症、充血性心力衰竭、脱水、大剂量利尿剂、急性大出血等）
├── 是 → 不适用后续流程，建议去医院找专业医生判断
└── 否
    └── Hb < 参考下限？（依据 WS/T 405 最新版本）
        ├── 否（Hb ≥ 参考下限）→ 停止，当前不满足贫血诊断
        │   提示排除干扰因素后重新人工评估
        └── 是（存在贫血）→ 进入形态学分类
            ├── MCV & MCH & MCHC 均低？（小细胞低色素）
            │   ├── 否 → 跳至【特别提示路径】
            │   └── 是
            │       └── 条款 2～9 满足 ≥ 2 条？
            │           ├── 是 → ✅ 确诊缺铁性贫血（IDA）
            │           └── 否 → 跳至【特别提示路径】
            └── 【特别提示路径】
                ├── SF 有值且 SF < 14 μg/L？
                │   ├── 是 → ⚠️ 高度提示 IDA，请进一步确认
                │   └── 否 → ❌ 证据不足，不能诊断 IDA
```

### 7.2 诊断等级定义

| 等级 | 名称 | 含义 |
|------|------|------|
| 确诊 | 可诊断为缺铁性贫血（IDA） | 小细胞低色素 + 条款 2~9 中 ≥ 2 条阳性 |
| 提示 | 高度提示缺铁性贫血 | SF < 14 μg/L 但未达确诊标准 |
| 阴性 | 证据不足，不能诊断 | 不满足上述任一条件 |
| 不适用 | 存在前置条件 | 血容量变化等情况，系统无法自动判断 |

### 7.3 计算链路伪代码

完整的 Python 伪代码实现详见附录 A。关键逻辑摘要：

```python
# 条款 2~9 中阳性数量（None 表示未检查，不计入）
supporting_conditions = [c2, c3, c4, c5, c6, c7, c8, c9]
positive_count = sum(1 for c in supporting_conditions if c is True)

if clinical.get("precondition", False):
    verdict = "不适用后续流程"
elif not c0:  # Hb 未低于参考下限
    verdict = "不满足贫血诊断"
elif c1 and positive_count >= 2:  # 小细胞低色素 + ≥2 条阳性
    verdict = "可诊断为缺铁性贫血（IDA）"
elif c7 is True:  # SF < 14
    verdict = "高度提示缺铁性贫血"
else:
    verdict = "证据不足，不能诊断缺铁性贫血"
```
